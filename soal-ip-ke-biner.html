6. konversikan bilamgan biner tersebut dalam bentuk ip address format decimal 11000011.11001011.01111101.00011001​


Jawaban singkat:

195.203.125.25

Penjelasan tambahan:

Perhitungan konversi biner ke decimal:

11000011 = 1x2⁷+1x2⁶+0x2⁵+0x2⁴+0x2³+0x2²+1x2¹+1x2⁰

11000011 = 128+64+0+0+0+0+2+1

11000011 = 195

Jadi 11000011 (biner) = 195 (decimal)

Perhitungan konversi biner ke decimal:

11001011 = 1x2⁷+1x2⁶+0x2⁵+0x2⁴+1x2³+0x2²+1x2¹+1x2⁰

11001011 = 128+64+0+0+8+0+2+1

Jadi 11001011 (biner) = 203 (decimal)

Perhitungan konversi biner ke decimal:

01111101 = 0x2⁷+1x2⁶+1x2⁵+1x2⁴+1x2³+1x2²+0x2¹+1x2⁰

01111101 = 0+64+32+16+8+4+0+1

01111101 = 125

Jadi 01111101 (biner) = 125 (decimal)

Perhitungan konversi biner ke decimal:

00011001​ = 0x2⁷+0x2⁶+0x2⁵+1x2⁴+1x2³+0x2²+0x2¹+1​x2⁰

00011001​ = 0+0+0+16+8+0+0+1

00011001 = 25

Jadi 00011001 (biner) = 25 (decimal)

11. Berapa nomor IP dari biner berikut 00011001. 01100100.00100001.00100001 adalah ​


Jawaban singkat:

25.100.33.33

Penjelasan panjang:

Bila maksudnya adalah nilai desimal dari IP biner tersebut, maka konversinya (biner ke desimal) bisa dilakukan sebagai berikut:

00011001 = 0x2⁷+0x2⁶+0x2⁵+1x2⁴+1x2³+0x2²+0x2¹+1x2⁰ = 25

Jadi 00011001 (biner) = 25 (desimal)

01100100 = 0x2⁷+1x2⁶+1x2⁵+0x2⁴+0x2³+1x2²+0x2¹+0x2⁰ = 100

Jadi 01100100 (biner) = 100 (desimal)

00100001 = 0x2⁷+0x2⁶+1x2⁵+0x2⁴+0x2³+0x2²+0x2¹+1x2⁰ = 33

Jadi 00100001 (biner) = 33 (desimal)

00100001 = 0x2⁷+0x2⁶+1x2⁵+0x2⁴+0x2³+0x2²+0x2¹+1x2⁰ = 33

Jadi 00100001 (biner) = 33 (desimal)

Maka kesimpulannya yaitu

00011001.01100100.00100001.00100001 (biner) = 25.100.33.33 (desimal)

22. mengapa IP Address dikatakan terdiri dari 32 bit biner​#plis jawab kak


Jawaban:

IP address adalah sebuah sistem pengalamatan unik setiap host yang terkoneksi ke jaringan berbasis TCP/IP. IP address bisa dianalogikan seperti sebuah alamat rumah. Ketika sebuah datagram dikirim, informasi alamat inilah yang menjadi acuan datagram agar bisa sampai ke device yang dituju. IP Address terbagi dalam 2 versi, IPv4 dan IPv6. Sebuah IP address versi 4 atau IPv4 terbentuk dari 32 binary bits. Dari 32 binary bits tersebut terbagi lagi menjadi 4 octet (1 octet = 8 bits). Nilai tiap oktet diatara 0 sampai 255 dalam format desimal, atau 00000000 - 11111111 dalam formal binary. Setiap octet dikonversi menjadi desimal dan dipisahkan oleh tanda titik (dot). Sehingga format akhir IP address biasanya berupa angka desimal yang dipisahkan dengan tanda titik, contohnya 172.16.254.1.

30. berapa jumlah oktal biner dalam satu rangkaian ip address versi 4​


Jawaban:

Dalam satu rangkaian alamat IP versi 4 (IPv4), terdapat empat bagian oktal yang disebut oktet. Setiap oktet direpresentasikan dalam bentuk bilangan biner dengan panjang 8 bit. Jadi, jumlah total bit dalam alamat IP versi 4 adalah 4 oktet * 8 bit/oktet = 32 bit.

Sebagai contoh, alamat IP versi 4 "192.168.0.1" dapat diuraikan menjadi biner sebagai berikut:

- 192: 11000000

- 168: 10101000

- 0: 00000000

- 1: 00000001

Total bitnya adalah 32 (8 bit + 8 bit + 8 bit + 8 bit).

37. Ip address versi 4 (ipv4) merupakan kumpulan bilangan biner yang terdiri dari…


Jawaban & Penjelasan:

Alamat IP terdiri dari 32 bit.

Alamat IP terdiri dari 32 bit, sering ditampilkan sebagai 4 oktet angka dari 0-255 yang direpresentasikan dalam bentuk desimal, bukan dalam bentuk biner. Misalnya, alamat IP: 168.212.226.204 dalam bentuk biner adalah 10101000.11010100.11100010.11001100.

Internet Protocol versi 4 (IPv4) mendefinisikan alamat IP sebagai angka 32-bit. Namun, karena pertumbuhan Internet dan menipisnya alamat IPv4 yang tersedia, versi baru IP (IPv6), menggunakan 128 bit untuk alamat IP, dikembangkan pada tahun 1995, dan distandarisasi pada bulan Desember 1998.

Ukuran awalan, dalam bit, ditulis setelah miring. Ini disebut "notasi garis miring". Ada total 32 bit dalam ruang alamat IPv4. Misalnya, jika sebuah jaringan memiliki alamat "192.0.2.0/24", angka "24" mengacu pada berapa banyak bit yang terkandung dalam jaringan. Ini berarti alamat host tunggal. Itu disebut notasi CIDR. 192.168.0.1 mewakili IP dan /32 mewakili jumlah bit dalam mask.
