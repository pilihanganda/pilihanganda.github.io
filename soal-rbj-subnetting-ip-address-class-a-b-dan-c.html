1. Hitunglah subnetting IP Address Kelas C dari IP 192.10.10.10 a. Jumlah Subnet​


Untuk melakukan subnetting pada alamat IP Kelas C 192.10.10.10, kita akan membagi alamat IP ini menjadi beberapa subnet. Jumlah subnet yang dapat dibuat tergantung pada seberapa banyak bit host yang akan dialokasikan ke dalam subnet dan berapa banyak bit yang akan digunakan untuk subnetting.

Pada alamat IP Kelas C, biasanya, 24 bit pertama (dari bit 0 hingga bit 23) adalah bagian dari alamat jaringan, dan 8 bit terakhir (dari bit 24 hingga bit 31) adalah bagian dari alamat host. Namun, kita dapat membagi alamat IP ini lebih lanjut dengan meminjam beberapa bit dari alamat host untuk digunakan sebagai subnet.

Misalnya, jika kita ingin menggunakan 4 bit untuk subnetting, maka kita akan meminjam 4 bit dari alamat host, sehingga tersisa 4 bit untuk host. Ini akan menghasilkan 2^4 = 16 subnet yang berbeda.

Jadi, jumlah subnet yang dapat dibuat adalah 16.

3. Hitunglah Subnetting IP address 192.168.10.10/27​


Untuk melakukan subnetting pada IP address 192.168.10.10/27, pertama-tama kita perlu mengetahui beberapa informasi dasar tentang subnet mask.

Dalam notasi CIDR, /27 menunjukkan jumlah bit yang digunakan untuk mask subnet. Jadi, /27 berarti bahwa 27 bit diperuntukkan untuk mask subnet, sedangkan sisa 5 bit diperuntukkan untuk host.

Sekarang kita dapat menghitung beberapa informasi penting dari subnet ini:

Jumlah subnet yang tersedia: 2^5 = 32Jumlah host yang tersedia per subnet: 2^(32-27) - 2 = 30Network ID: 192.168.10.8 (192.168.10.10 & 255.255.255.224)Broadcast ID: 192.168.10.31 (192.168.10.10 | 0.0.0.31)Rentang IP yang tersedia: 192.168.10.9 - 192.168.10.30

Jadi, dengan subnet mask /27, kita dapat membuat 32 subnet yang masing-masing dapat menampung 30 host. Subnet ini diawali dari alamat IP 192.168.10.8 dan diakhiri dengan alamat IP 192.168.10.31.

Jika anda butuh subnetting IP address yang lain dan jenis subnetting yang lain, silahkan tanyakan kembali kepada saya.

5. diketahui ip address 192.168.10.100 tuliskan a. jumlah subnet b. ip validc. ip broadcast​


Untuk menjawab pertanyaan Anda, diperlukan informasi tambahan tentang subnet mask yang digunakan dalam jaringan tersebut. Subnet mask diperlukan untuk mengidentifikasi subnet, alamat IP yang valid, dan alamat IP broadcast.

Misalkan kita anggap subnet mask yang digunakan adalah 255.255.255.0 (CIDR: /24), berikut adalah jawaban untuk setiap pertanyaan:

a. Jumlah Subnet:

Jumlah subnet dapat ditentukan dengan melihat jumlah bit yang digunakan untuk subnet pada subnet mask. Dalam kasus ini, subnet mask adalah 255.255.255.0, yang berarti ada 24 bit yang digunakan untuk subnet dan 8 bit yang digunakan untuk host. Dengan 24 bit subnet, jumlah subnet yang mungkin adalah 2^24 - 2 (2^24 mengurangi subnet yang terreservasi untuk alamat jaringan dan alamat broadcast). Jadi, jumlah subnet yang mungkin adalah 2^24 - 2 = 16.777.214 subnet.

b. Alamat IP Valid:

Alamat IP yang valid dalam subnet ini adalah alamat yang digunakan untuk host. Dalam kasus ini, alamat IP 192.168.10.100 adalah alamat host, jadi itu adalah alamat IP yang valid dalam subnet tersebut.

c. Alamat IP Broadcast:

Alamat IP broadcast adalah alamat yang digunakan untuk mengirim pesan ke semua host dalam subnet. Dalam kasus ini, alamat IP broadcast dapat ditentukan dengan mengatur semua bit host pada alamat IP menjadi 1. Jadi, alamat IP broadcast untuk subnet tersebut adalah 192.168.10.255.

11. Pembagian IP class dibedakan berdasarkan versinya, IPv4 dan IPv6. Pada IPv4, IP Address dibagi menjadi class A, B, C, D, E Kelas A, B, C jumlah host dan jumlah network bisa didefinisikan. Sedangkan kelas D dan E tidak bisa didefinisikan, kenapa? Jelaskan!


Jawaban:

Kelas IP Address

Guna memudahkan dalam pembagiannya maka IP address dibagi-bagi ke dalam kelas-kelas yang berbeda, yaitu sebagai berikut:

1) Kelas A

IP address kelasA terdiriatas 8 bit untuk network ID dan sisanya 24 bit digunakan untuk host ID, sehingga IP address kelas A digunakan untuk jaringan dengan jumlah host sangat besar. Pada bit pertama diberikan angka 0 sampai dengan 127.

Karakteristik IP Kelas A

Format : 0NNNNNNN.HHHHHHHH.HHHHHHHH.HHHHHHHH

Bit pertama : 0

Network ID : 8 bit

Hos tID : 24 bit

Oktat pertama : 0 – 127

Jumlah network : 126 (untuk 0 dan 127 dicadangkan)

Rentang IP : 1.x.x.x – 126.x.x.x

Jumlah IP address : 16.777.214

Contoh

IP address 120.31.45.18 maka :

Network ID = 120

Host ID = 31.45.18

Jadi, IP diatas mempunyai host dengan nomor 31.45.18 pada jaringan 120.

2) Kelas B

IP address kelas B terdiriatas 16 bit untuk network ID dan sisanya 16 bit digunakan untuk host ID, sehingga IP address kelas B digunakan untuk jaringan dengan jumlah host tidakterlalubesar. Pada 2 bit pertama, diberikanangka 10.

Karakteristik IP Kelas B

Format : 10NNNNNN. NNNNNNNN.HHHHHHHH.HHHHHHHH

Bit pertama : 10

Network ID : 16 bit

Host ID : 16 bit

Oktat pertama : 128 – 191

Jumlah network : 16.384

Rentang IP : 128.1.x.x – 191.255.x.x

Jumlah IP address : 65.534

Contoh

IP address 150.70.60.56 maka :

Network ID = 150.70

HostID = 60.56

Jadi, IP diatas mempunyai host dengan nomor 60.56 pada jaringan 150.70

3) Kelas C

IP address kelas C terdiri atas 24 bit untuk network ID dan sisanya 8 bit digunaka nuntuk host ID, sehingga IP address kelas C digunakan untuk jaringan berukuran kecil. Kelas C biasanya digunakan untuk jaringan Local Area Network atau LAN. Pada 3 bit pertama, diberikan angka 110.

Karakteristik IP Kelas C

Format : 110NNNNN.NNNNNNNN. NNNNNNNN.HHHHHHHH

Bit pertama : 110

Network ID : 24 bit

HostID : 8 bit

Oktat pertama : 192 – 223

Jumlah network : 2.097.152

Rentang IP : 192.0.0.x – 223.255.225.x

Jumlah IP address : 254

Contoh

IP address 192.168.1.1 maka :

Network ID = 192.168.1

Host ID = 1

Jadi, IP diatasmempunyai host dengannomor 1 padajaringan 192.168.1

Kelas D

IP address kelas D digunakan untuk keperluan IP multicasting. 4 bit pertama IP address kelas D di set 1110. Bit-bit berikutnya diatur sesuai keperluan multicast group yang menggunakan IP address ini. Dalam multicasting tidak dikenal network bit dan host bit.

Karakteristik

Format : 1110MMMM.MMMMMMMM.MMMMMMMM.MMMMMMMM

4 bit pertama : 1110

Bit multicast : 28 bit

Byte insial : 224-247

Deskripsi : kelas D adalah ruang alamat multicast (RFC 1112)

5) Kelas E

IP address kelas E tidak digunakan untuk umum. 4 bit pertama IP address ini de set 1111.

Karakteristik

Format : 1111RRRR.RRRRRRRR.RRRRRRRR.RRRRRRRR

4 bit pertama : 1111

Bit cadangan : 28 bit

Byte inisial : 248-255

Deskripsi : kelas E adalah ruang alamat yang di cadangkan untuk keperluan

19. uraikan mengenai konsep subnetting dari ip address​


Jawaban:

Subnetting adalah proses pemecahan jaringan menjadi beberapa subnet yang berbeda. Dengan subnetting, Anda dapat mengatur lalu lintas jaringan secara efisien dan mengurangi tingkat konflik antara perangkat jaringan. Subnetting juga memungkinkan Anda membagi jaringan Anda menjadi beberapa kelompok yang berbeda berdasarkan topologi jaringan, lokasi atau kebutuhan keamanan. Dengan subnetting, Anda dapat mengatur lalu lintas jaringan Anda dengan menggunakan alamat IP address. Alamat IP address yang dipartisi menjadi subnet dapat digunakan untuk membedakan antara jaringan yang berbeda. Subnetting memungkinkan Anda mengontrol lalu lintas jaringan dan memastikan bahwa lalu lintas jaringan tidak saling bertabrakan.

37. 2. ip address 100.100.100.100tentukan class :oktet network:oktet host:subner mask:ip network:ip host:ip bc :​


Alamat IP 100.100.100.100 adalah contoh alamat IP, dan kita dapat menganalisisnya sebagai berikut:

Class: Alamat IP ini adalah bagian dari kelas A, karena rentang alamat IP kelas A adalah 0.0.0.0 hingga 127.255.255.255.

Oktet Network: Pada alamat IP kelas A, oktet pertama (100) adalah bagian dari alamat jaringan.

Oktet Host: Tiga oktet berikutnya (100.100.100) adalah bagian dari alamat host.

Subnet Mask: Karena ini adalah alamat kelas A, subnet mask standarnya adalah 255.0.0.0.

IP Network: Alamat jaringan adalah 100.0.0.0 (dengan tiga oktet pertama dari alamat IP).

IP Host: Alamat host adalah 0.100.100.100 (dengan tiga oktet terakhir dari alamat IP).

IP Broadcast: Alamat broadcast dalam jaringan ini adalah 100.255.255.255 (dengan tiga oktet pertama tetap dan oktet terakhir diatur menjadi 255).
